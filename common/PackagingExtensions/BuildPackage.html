Build a Package

Overview

The most basic DNN skin configuration is comprised of one layout template. However, you can also create a master CSS file for all layout templates or one CSS file for each layout template.

Prerequisites

Steps

  1. Prepare your files in folders.
    • Layout templates and associated files

      Tip: You can provide a preview by including an image file with the same filename as your layout template or container; example, add a MyLayoutOrContainer.jpg preview for the MyLayoutOrContainer.ascx layout template or container.

      • Layout template files (*.ascx)
      • Preview of individual layout templates (*.png/jpg/etc.)
      • Document Type XML file for individual layout templates (*.doctype.xml)
      • CSS for individual layout templates (*.css)
      • CSS for the theme (skin.css)
      • Custom attributes of tokens (skin.xml)
    • Containers and associated files
      • Container files (*.ascx)
      • Preview of individual containers (*.png/jpg/etc.)
      • CSS for individual containers (*.css)
      • Custom attributes of tokens (container.xml)
    • Modules and associated files
    • Folders
      • css - All CSS required by layout templates and containers.
      • fonts - All web fonts used by the theme.
      • images - All image files used by the theme.
      • js - All JScript files used by the theme, such as bootstrap.min.js and common.js.
      • menu - All desktop menu files for the skin.
      • mobmenu - All mobile menu files for the skin.
    • License/Release Notes

      These files are HTML-formatted files:

      • MyContainerLicense.txt
      • MyContainerReleaseNotes.txt
      • MyPackageLicense.txt
      • MyPackageReleaseNotes.txt
      • MySkinLicense.txt
      • MySkinReleaseNotes.txt

      Tip: In the license files, you can include special offers, including a call to action and other details.

      Reminder: Include the version number of your extension in the release notes.

  2. Create the DNN manifest.

    Note: Containers must be defined under a separate section, even though it would be installed as part of the theme.

    Example of a portion of the DNN manifest of a theme:

    <package name="..." type="Skin" version="..."> ... <license src="MySkinLicense.txt" /> <releaseNotes src="MySkinReleaseNotes.txt" /> ... <components> <component type="File"> <files> <file> <name>MyPackageLicense.txt</name> </file> <file> <name>MyPackageReleaseNotes.txt</name> </file> </files> </component> <component type="Skin"> <skinFiles> <basePath /> <!-- Target base folder for the component installation. Relative to the root of the DNN installation. --> <skinName>MyThemeName</skinName> <!-- Create a skinFile section for every CSS file in your theme. --> <skinFile> <path>css</path> <name>skin.css</name> </skinFile> ... <!-- Create a skinFile section for every font used in your theme. --> <skinFile> <path>fonts</path> <name>NameOfFont.ttf</name> </skinFile> ... <!-- Create a skinFile section for every other file used in your theme, such as images. --> <skinFile> <path>images</path> <name>MyImage.png</name> </skinFile> ... </skinFiles> </component> ... </components> </package> <package name="..." type="Container" version="..."> ... <license src="MyContainerLicense.txt" /> <releaseNotes src="MyContainerReleaseNotes.txt" /> ... <components> <component type="Container"> <containerFiles> <basePath /> <!-- Target base folder for the component installation. Relative to the root of the DNN installation. --> <containerName>MyThemeName</containername> <!-- Create a containerFile section for every container file in your theme. --> <containerFile> <name>MyContainer.ascx</name> </containerFile> <containerFile> <name></name> </containerfile> ... </containerFiles> </component> ... </components> </package>

    Note: Save the manifest file(s) (*.dnn*) in the base folder of your package.

  3. Zip up your files, preserving the folder structure.
