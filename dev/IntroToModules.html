Introduction to DNN Modules

Overview

A DNN module is custom code that extends the functionality of the DNN platform. A module can:

  • generate dynamic content,
  • display content from another source,
  • edit content,
  • manage data,
  • change database configuration,
  • provide administrative functions, or
  • provide other functionality.

Components of a Module

A module includes the following:
  • References\*.dll are libraries required by the module. Example: The DotNetNuke.dll is required in all modules; it provides additional functionality, such as authentication and user information.
  • Components\FeatureController.cs contains the list of DNN interfaces that the module implements.
  • The packages.config file specifies the recommended .NET Framework version for the module.
  • Data\DataProvider.cs defines the abstract data provider's methods to access data. The abstract data provider hides the details of the concrete data provider, so that the module can access the data, whether the concrete data provider is a SQL database or an Oracle database.

    The actual implementations (concrete data providers) are stored in the Provider\DataProviders folder. A module can use multiple concrete data providers.

    Note: This is just one of several ways to access data in DNN.

  • The Providers folder contains definitions of a href="http://www.dnnsoftware.com/wiki/providers">providers required by the module.

    Example: The subfolder SQLDataProvider contains three files:

    • 00.00.01 SQLDataProvider contains the SQL scripts that create tables and stored procedures required by the module to access the data. Run when the module is installed.
    • Uninstall.SqlDataProvider contains the SQL scripts that delete tables and stored procedures used by the module. Run when the module is uninstalled.
    • SQLDataProvider.cs defines the concrete data provider class's methods to access data, as well as set the value of SqlConnection.ConnectionString.

  • (Optional) App_LocalResources\*.resx are string tables that contain localized or translated terms.
  • (Optional) Documentation\Documentation.html describes the module functionality and is provided to the user during module installation.
  • (Optional) ReleaseNotes.txt lists the changes for the current version of the module and is also provided during installation.
  • (Optional) License.txt is displayed to the user during module installation.
  • (Optional) The module.css defines how the module should be displayed. If the theme on the webpage includes a container for the pane that the module is associated with, the container's CSS overrides the module's CSS.
  • The *.dnn file is the DNN manifest, which is an XML file that specifies where the components of the module must be installed. The filename must match the name of the module.
  • Properties\Assembly.cs contains information about the module package.
  • The .build folder contains files required for MSBuild to compile the module and to assemble the installation file.
