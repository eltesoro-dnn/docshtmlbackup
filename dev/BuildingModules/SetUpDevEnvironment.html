Set up Your Development Environment

Overview

To set up a local development environment to create DNN modules:

  1. Determine what DNN version to install.
  2. Install the following:
    • IIS with ASP.NET extensions
    • .NET Framework
    • SQL
    • Visual Studio 2015

    Check the DNN 8.0 Requirements for compatible versions.

  3. Install DNN.

Note: These instructions use DNNDEV.ME as the domain of your local DNN installation. DNNDEV.ME is a registered domain which points to the loopback address of 127.0.0.1, so it will always resolve locally.

  1. Choose a DNN product.
    • The DNN Platform is the free open-source package.
    • Evoq (both Content and Engage) adds custom functionality over the DNN Platform.
  2. Check the requirements and install the appropriate versions for the chosen DNN version.
  3. Download the DNN Platform.

    Note: For module development, you won't need the source files.

  4. Set file permissions for the DNN installation folder to Full or Modify.
    • In Windows Explorer, right-click on the folder where you installed DNN, and choose Properties.
    • Go to the Security tab.
      Note: If you are using Windows XP, go to Tools > Options and uncheck Use Simple File Sharing.
    • Give Full or Modify permissions to the user account that is used to run your website. The account name differs, depending on your IIS and Windows version:
      If your IIS version is and your OS is then you must give Full or Modify permissions to
      IIS 5
      IIS 5.1       		Windows 2000
      Windows XP       		localmachine\ASPNET
      IIS 6
      IIS 7       		Windows 2003
      Windows Vista
      Windows Server 2008       		localmachine\Network Service
      IIS 7.5 Windows 2008 R2
      Windows 7       		the Network Service account, or
      the IIS AppPool\AppPoolName, which is the name of the automatically generated app.
  5. Create a SQL database.
    1. In SQL Server, create a new database.
    2. Create a SQL user to access the database.
      • Use SQL authentication.
      • Turn off the Enforce Password requirement.
      • Give the new SQL user DB Owner and Public access to the new database.
      • Remember the username and password, because they will be required in the DNN Installation Wizard.
  6. Set up IIS.
    • Create a new website and point to the folder where you installed the DNN Platform.

      Note: If you use an existing IIS website, add the HOST binding for DNNDEV.ME in IIS.

    • For IIS 5 or IIS 6, verify that ASP.NET 2.0 is selected on the ASP.NET tab.
  7. Run the DNN Installation Wizard.
    1. Open your website (http://dnndev.me) in a browser.
    2. In the installation wizard, choose Typical and follow the prompts to Database Configuration.
    3. In Database Configuration,
      1. Choose SQL 2005/2008.
      2. Uncheck integrated authentication.
      3. Use (local) as the name of your SQL Server.
      4. Enter your database name.
      5. Enter the username and password of the SQL user you created previously.
      6. If creating modules for your own websites, set objectQualifier to blank.
        If creating modules for sale, set objectQualifier to dnn to prepend dnn_ to all DNN-generated objects, such as tables and stored procedures. This practice is recommended to avoid naming conflicts in your clients' databases.
    4. Follow the prompts to the end of the wizard.
      Note:Remember the HOST and ADMIN passwords that you provide.
  8. Modify the web.config file.
    • ShowMissingKeys = TRUE

      If TRUE,

      • Any localized text in DNN will be displayed with an "L" in front of it to help with translating text in your module.
      • DNN displays an error if the localized version of a string is missing from the resource files. The resource files are XML files that contain the localized string tables.
